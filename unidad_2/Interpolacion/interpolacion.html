1. Interpolación Lineal

Ejercicio 1.1
Estimación del tiempo de procesamiento en algoritmos paralelos

Un equipo de investigación analiza el desempeño de un algoritmo de procesamiento distribuido:

Nodos 4 10
Tiempo (s) 22.5 11.2

Enunciado:

  • Dados los puntos (4 nodos, 22.5 s) y (10 nodos, 11.2 s)
  • Estimar el tiempo cuando se emplean 7 nodos usando interpolación lineal
  • Aplicar fórmula: \( y = y_0 + \frac{y_1 - y_0}{x_1 - x_0}(x - x_0) \)

Solución

Fórmula de interpolación lineal:

\[ y = 22.5 + \frac{11.2 - 22.5}{10 - 4}(7 - 4) \]

\[ y = 22.5 + \frac{-11.3}{6} \cdot 3 \]

Tiempo estimado
16.85 s
Reducción
25.1%
Conclusión: Con 7 nodos, el tiempo estimado de ejecución es aproximadamente 16.85 segundos, lo cual refleja una reducción proporcional del tiempo conforme aumenta el paralelismo.

2. Interpolación de Lagrange

Ejercicio 2.1
Rendimiento energético de procesadores
Frecuencia (GHz) 2.0 2.5 3.0
Eficiencia (MIPS/Watt) 65 70 60

Solución

Polinomios de Lagrange:

\[ L_0(x) = 2(x-2.5)(x-3.0) \]

\[ L_1(x) = -4(x-2.0)(x-3.0) \]

\[ L_2(x) = 2(x-2.0)(x-2.5) \]

P(2.3 GHz)
69.8 MIPS/W
L₀(2.3)
0.28
L₁(2.3)
0.84
L₂(2.3)
-0.12
Conclusión: A 2.3 GHz, se estima un rendimiento de aproximadamente 69.8 MIPS/Watt.
lagrange_ejemplo.R 
# Interpolación de Lagrange - Ejercicio 2.1
x <- c(2.0, 2.5, 3.0)
y <- c(65, 70, 60)

# Función para calcular polinomios de Lagrange
lagrange_interp <- function(x_vals, y_vals, x_target) {
  n <- length(x_vals)
  result <- 0
  
  for (i in 1:n) {
    # Calcular L_i(x_target)
    L_i <- 1
    for (j in 1:n) {
      if (j != i) {
        L_i <- L_i * (x_target - x_vals[j]) / (x_vals[i] - x_vals[j])
      }
    }
    result <- result + y_vals[i] * L_i
  }
  return(result)
}

# Estimación para 2.3 GHz
estimacion <- lagrange_interp(x, y, 2.3)
cat("Rendimiento estimado a 2.3 GHz:", round(estimacion, 1), "MIPS/Watt\n")

3. Diferencias Divididas de Newton

Ejercicio 3.1
Tiempo de cómputo en algoritmos de clasificación
Tamaño muestra 1000 2000 5000
Tiempo (s) 0.8 1.6 4.5

Solución

Diferencias divididas:

\[ f[x_0,x_1] = 0.0008 \]

\[ f[x_1,x_2] = 0.000967 \]

\[ f[x_0,x_1,x_2] = 4.17 \times 10^{-8} \]

P(3000)
2.48 s
Polinomio
P(x) = 0.8 + 0.0008(x-1000) + 4.17e-8(x-1000)(x-2000)
Conclusión: Se estima un tiempo de procesamiento de 2.48 segundos para 3000 muestras.

5. Splines Cúbicos

Ejercicio 5.1
Suavizado de datos de temperatura
Hora 0 3 6 9 12
Temperatura (°C) 18 20 23 22 19

Solución

Condiciones del spline cúbico natural:

\[ S_i(x) = a_i + b_i(x-x_i) + c_i(x-x_i)^2 + d_i(x-x_i)^3 \]

\[ S''(0) = S''(12) = 0 \]

Método
Spline Cúbico Natural
Aplicación
Calibración de sensores
Ventaja
Suavidad C²
Conclusión: El spline cúbico natural produce una curva que suaviza los puntos medidos, útil en calibración de sensores y control térmico en sistemas computacionales.

spline_cubico.R 
# Spline Cúbico Natural - Ejemplo
library(splines)

# Datos del servidor
usuarios <- c(100, 200, 300, 400)
tiempo <- c(120, 135, 180, 250)

# Crear spline cúbico natural
spline_natural <- splinefun(usuarios, tiempo, method = "natural")

# Estimación para 250 usuarios
estimacion_250 <- spline_natural(250)

cat("Tiempo estimado para 250 usuarios:", round(estimacion_250, 1), "ms\n")

# Generar puntos para graficar
x_seq <- seq(100, 400, length.out = 100)
y_seq <- spline_natural(x_seq)

# Mostrar resultados
cat("Valores del spline:\n")
for(i in seq_along(x_seq)) {
  if(i %% 10 == 0) {
    cat(sprintf("x=%.0f, y=%.1f\n", x_seq[i], y_seq[i]))
  }
}

6. Error de Interpolación

Teoría
Cota del Error de Interpolación

Error para polinomio de grado n:

\[ |f(x) - P_n(x)| \leq \frac{M_{n+1}}{(n+1)!} \prod_{i=0}^n |x - x_i| \]

donde \( M_{n+1} \) es el máximo de \( |f^{(n+1)}(x)| \) en el intervalo.

Comparación de métodos:

  • Interpolación lineal: Rápida y simple, error proporcional a \((x-x_i)^2\)
  • Lagrange/Cuadrática: Mayor precisión en zonas suaves
  • Splines cúbicos: Menor error global y continuidad C²
Conclusión: El estudio del error de interpolación es esencial para evaluar la calidad de las aproximaciones numéricas y seleccionar el método más eficiente en problemas de ingeniería estadística e informática.
