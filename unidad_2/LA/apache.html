Evaluación de Plataformas Web

Contexto del Estudio

En la actualidad, la evaluación del rendimiento de aplicaciones web y plataformas digitales es fundamental para garantizar una experiencia de usuario óptima. Apache JMeter es una herramienta de código abierto ampliamente utilizada para realizar pruebas de carga, estrés y rendimiento, permitiendo simular múltiples usuarios que acceden simultáneamente a un sitio.

Google Lighthouse es una herramienta automatizada que evalúa la calidad de páginas web en aspectos clave como rendimiento, accesibilidad, buenas prácticas, SEO y experiencia de usuario. Complementa las pruebas de carga proporcionando una visión más integral del rendimiento real que percibe el usuario final.

Objetivo del Estudio

Evaluar el rendimiento y capacidad de respuesta de YouTube y Aula Virtual UNA Puno mediante pruebas con Apache JMeter y Google Lighthouse. También se aplican métodos de optimización numérica como interpolación lineal y Regula Falsi para analizar y mejorar los tiempos de respuesta y la eficiencia de cada plataforma.

Análisis con Métodos Numéricos

Se aplican métodos numéricos para determinar los límites operativos óptimos de cada plataforma:

Interpolación Lineal: Para encontrar usuarios dado un tiempo promedio objetivo

\[ U_{target} = U_1 + \frac{(A_{target} - A_1) \cdot (U_2 - U_1)}{A_2 - A_1} \]

Regula Falsi: Para encontrar usuarios dado un porcentaje de error objetivo

\[ c = b - \frac{f(b) \cdot (b - a)}{f(b) - f(a)} \]

optimizacion_rendimiento.R 
# Análisis de Optimización - YouTube
usuarios <- c(100, 500, 1000)
average <- c(39777, 57318, 131565)
error <- c(0, 13.6, 60.8)

# Interpolación lineal para encontrar usuarios dado un average objetivo
interpolacion_average <- function(average_target, usuarios, average) {
  for(i in 1:(length(usuarios)-1)) {
    if(average_target >= average[i] && average_target <= average[i + 1]) {
      U1 <- usuarios[i]
      U2 <- usuarios[i + 1]
      A1 <- average[i]
      A2 <- average[i + 1]
      U_target <- U1 + (average_target - A1) * (U2 - U1) / (A2 - A1)
      return(U_target)
    }
  }
  return(NA)
}

# Método Regula Falsi para encontrar usuarios dado un error objetivo
regula_falsi <- function(error_target, usuarios, error, tol = 1e-3, maxiter = 100) {
  f <- function(U) {
    for(i in 1:(length(usuarios)-1)) {
      if(U >= usuarios[i] && U <= usuarios[i + 1]) {
        return(error[i] + (U - usuarios[i]) * 
               (error[i+1] - error[i]) / (usuarios[i+1] - usuarios[i]) - error_target)
      }
    }
    return(NA)
  }
  
  a <- min(usuarios)
  b <- max(usuarios)
  fa <- f(a)
  fb <- f(b)
  iter <- 0
  
  while(iter < maxiter) {
    c <- b - fb * (b - a) / (fb - fa)
    fc <- f(c)
    
    if(abs(fc) < tol) return(c)
    
    if(fc * fa < 0) {
      b <- c
      fb <- fc
    } else {
      a <- c
      fa <- fc
    }
    iter <- iter + 1
  }
  return(c)
}

# Cálculos para objetivos específicos
average_obj <- 50000
error_obj <- 5

U_avg <- interpolacion_average(average_obj, usuarios, average)
U_err <- regula_falsi(error_obj, usuarios, error)

cat("=== RESULTADOS DE OPTIMIZACIÓN ===\n")
cat("Usuarios según Average de 50000 ms:", round(U_avg), "\n")
cat("Usuarios según Error de 5%:", round(U_err), "\n")

# Recomendación operativa
cat("\n=== RECOMENDACIÓN OPERATIVA ===\n")
if (U_err < U_avg) {
  cat("Operar con máximo", round(U_err), "usuarios para garantizar calidad\n")
  cat("Entre", round(U_err), "-", round(U_avg), "usuarios: tiempo aceptable pero con errores\n")
  cat("Sobre", round(U_avg), "usuarios: degradación severa del rendimiento\n")
}

Conclusiones para Aula Virtual

  • Requiere mejoras urgentes de infraestructura
  • Considerar implementación de balanceadores de carga
  • Optimización de base de datos y consultas
  • Cache de contenido estático y dinámico

Conclusiones para YouTube

  • Mayor tolerancia a la carga que Aula Virtual
  • Buen rendimiento en métricas de accesibilidad
  • Necesita mejoras en Core Web Vitals (LCP, INP, CLS)
  • Optimización de recursos multimedia
Conclusión General: Los métodos numéricos utilizados permiten estimar el punto crítico donde el rendimiento de cada plataforma comienza a deteriorarse. YouTube muestra mayor tolerancia a la carga, mientras que Aula Virtual presenta saturación temprana debido a limitaciones de infraestructura. La combinación de Apache JMeter con técnicas como interpolación y Regula Falsi permite obtener análisis más precisos y útiles para la toma de decisiones.
